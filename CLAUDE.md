# ひだまり防災ボード

家族4世帯（関西1・関東3）＋職場ライト2拠点の「在宅避難」を支える、家族専用の防災PWA。
備蓄の見える化・ハザード確認・安否確認を、非エンジニアを含む家族7人（＋犬2匹）がスマホで使う。

## 必読ドキュメント（毎回ここから読む）

- `docs/requirements.md` — 要件定義（何を作るか・フェーズ計画・スコープ外）
- `docs/tech-spec.md` — 技術仕様（スタック・データモデル・外部API一覧・制約）
- `docs/safety-policy.md` — 安全方針（個人情報・免責・災害時挙動）
- `reference/hidamari-bousai-phase1.jsx` — フェーズ1の参考実装（動作確認済み）。
  計算係数・画面構成・日本語文言はこのファイルを「正」とする。

## 絶対ルール

1. **個人情報をコードに書かない。** 実住所・勤務先・氏名・最寄駅・ペット名を、
   コード／ドキュメント／コミットメッセージ／ログ／テストデータに書かない。
   拠点は匿名ID（base_a〜base_f）で扱い、実データはアプリのUIから入力してDBにのみ保存する。
2. `INITIAL_DATA_PRIVATE.md`（存在する場合）と `.env` は**絶対にコミットしない**。
   `.gitignore` に登録済み。この行を消さない。
3. **購入の自動実行はしない。** アラート→Amazonリンクを開くまでが本アプリの仕事。
4. 基本方針は**在宅避難**。ダッシュボードの単位は「在宅避難◯日分」。
5. 安否確認は状態（無事／職場に待機／帰宅開始＋ペット対応済み）のみを送る。
   **位置情報は取得も送信もしない。**

## 作業の進め方

- フェーズ順（1→2→3）に作る。フェーズをまたぐ先行実装をしない。
- 大きな変更・新しい依存の追加の前に、短い計画を提示して承認を得る。
- モバイルファースト（iPhone Safari・幅375px基準）。UIはすべて日本語。
- 各フェーズ完了時：ローカル起動手順をREADMEに追記し、スマホ実機での確認を促す。
- APIキー等の秘密情報は `.env` に置き、クライアントに露出させてよいのは
  Supabaseのanonキー・VAPID公開鍵などの公開前提の値のみ。

## 技術方針（詳細は tech-spec）

- PWA：オフライン動作・ホーム画面インストール・Web Push（iOSはホーム画面追加が条件）
- 認証つきDB：Supabase想定。Row Level Securityで世帯ごとの公開範囲を制御
- 無料〜低コスト枠で運用できる構成を維持する
