# 要件定義書 — ひだまり防災ボード

版 1.0（2026-07-17）／出典：家族との設計対話で確定した仕様。
実地名・実住所は本書に書かない（別管理の INITIAL_DATA_PRIVATE.md にのみ記載。非コミット）。

## 1. 目的と基本方針

- 南海トラフ地震（関西側）・首都直下地震（関東側）を主対象に、家族4世帯＋職場2拠点の**在宅避難**を成立させる。
- 在宅避難は無条件ではなく**「スイッチ付き」**：拠点ごとに中断条件を1つ定義し、拠点カードの先頭に大きく表示する。
- 合言葉：「避難所は住む場所ではなく、水と情報を取りに行く場所」。在宅方針でも最寄り避難拠点の情報は消さない。
- 帰宅困難が想定される勤務者には「歩いて帰る装備」より**「帰らない判断を支える仕組み」**（職場待機＋安否確認）を提供する。

## 3. 機能要件

### F1 家族・拠点管理（フェーズ1〜、属性拡張はフェーズ2）
- 拠点CRUD、種別（住居／職場ライト）、世帯構成（大人数・犬数）。
- 建物属性：構造種別／居住階／築年3択（1981年6月以降・以前・不明）／オートロック停電時挙動（解錠・施錠・保持・不明）。
- **要確認バッジ**：未確認項目を拠点カードに表示し、確認方法のヒントを添える。「不明」のままでも運用が止まらないこと。
- 敷地内・隣接リソース欄（畑・貯湯タンク・井戸候補など自由記述＋種別タグ）。
- 勤務パターン登録 → **お留守番リスク欄**：犬のいる拠点で全員不在になりうる曜日・時間帯を自動表示し、対策チェックリスト（自動給水器・給餌器と電池、犬の安全ゾーン、緊急時に犬を見てくれる人の連絡先）を出す。

### F2 備蓄プランナー（フェーズ1）
- 期間：7日／14日／30日。
- 係数（確定値。参考実装 `reference/hidamari-bousai-phase1.jsx` と一致させる）：
  - 大人1人1日＝水3L・主食3食・携帯トイレ5回
  - カセットボンベ＝0.45本/人日（大人2人で約6本/週）
  - 犬1匹1日＝水0.5L・フード1日分・ペットシーツ3枚（目安。将来は体重入力で補正）
- 省スペースモード（狭い住戸向け）：生活用水は汲み置き＋給水袋で代替、食料はローリングストック、分散収納の提案。**共用廊下・ベランダ避難ハッチ周りへの収納は提案しない。**
- アレルギー・好みタグはフェーズ2以降のオプション。

### F3 在庫管理（フェーズ1）
- 品名／算入先（水・主食・トイレ・ボンベ・犬フード・シーツ・その他）／数量／期限／保管場所／商品URL／JAN（任意）。
- 消費ボタン（数量減）→ 不足はアラートに現れる。
- 大地震後は「排水管の確認までトイレを流さない」等の家族ルールをヒント表示（マンション拠点）。
- バーコード読み取り・レシートOCRはフェーズ2以降のオプション（必須にしない）。

### F4 見える化ダッシュボード（フェーズ1）
- 拠点別リング「**在宅避難◯日分**」＝ 水・主食・トイレの最少日数（在庫÷日需要）／目標日数。
- カテゴリ別ミニバー、要対応件数チップ。
- 中断スイッチ（拠点ごとの1条件）をカード先頭に表示（データ項目はF1、表示はフェーズ1から）。

### F5 アラート → Amazon（フェーズ1）
- 種別：期限30日前／期限切れ／目標不足。
- リンクは3段構え：①保存済み商品URL → ②JANでAmazon検索 → ③商品名で検索。
- **自動注文・自動補充・自動配送は行わない（確定方針）。**

### F6 ハザード表示（フェーズ2）
- 住所→ジオコーディング→タイル重ね：震度（J-SHIS）／液状化／洪水・内水／土砂。
- 津波レイヤーは拠点ごとにON/OFF（内陸拠点は初期OFF）。
- 浸水想定深 × 居住階の自動判定（例：2階なら垂直避難で足りるか）。
- 避難先の**2層表示**：指定緊急避難場所＝行政界を無視した純粋な最寄り／指定避難所＝自治体割当てで区別。
- 拠点ごとに自治体公式Web地図等の参照リンクを登録できる。
- 各データの出典表示・利用規約順守（tech-spec参照）。

### F7 帰宅ルート・職場待機（フェーズ2）
- 拠点フィールド：最寄駅・路線・駅までの手段と所要時間。
- **ルート3段構え**：通常／部分復旧（代替路線）／徒歩フル。世帯単位で共通ルートに統一できること。
- 運行情報表示（公共交通オープンデータ。対応路線のみ、非対応は非表示）。
- 職場ライト拠点：職場ロッカー在庫（F3流用）、会社の待機体制チェック（待機ルール・3日分備蓄の有無）、帰宅開始モード（徒歩ルート＋沿道支援拠点のオフライン地図）。

### F8 安否確認・訓練モード（フェーズ3）
- 地震連動：P2P地震情報を監視し、登録拠点の地域で震度5弱以上（しきい値可変）→ 該当拠点メンバーに安否確認プッシュ、他拠点に見守り通知。
- 手動開始ボタン（誰でも・停電や地震以外の災害用）。
- ステータス：**無事です／職場に待機します／帰宅開始します**＋ペット対応済みチェック。
- 地図ピン：緑＝無事／赤＝助けが必要／グレー＝未応答。
- オフライン：集合場所・家族連絡カード・持ち出しリストを事前キャッシュ。送信失敗は回復時に自動再送。
- 訓練モード：模擬速報で本番と同一UI（画面に常時「訓練」帯）。標準シナリオ「平日昼・base_c夫婦とも出社・犬留守番」。9/1・3/11にリマインド。年1回の在宅避難ドリル項目：ブレーカー位置・携帯トイレ試用・電源残量点検・太陽光自立運転切替・車両給電の実地練習。
- **位置情報は扱わない。**

### F9 通知（フェーズ1後半〜）
- Web Push：期限・不足・地震（F8）・定期点検（電源充電、燃料半分、訓練日）。
- iOSはホーム画面追加PWAが条件である旨をオンボーディングで案内。
- 通知音のカスタムはしない（確定）。深夜帯は非常時以外の通知を抑制。

### F10 見守り・贈る（フェーズ3）
- 家族間の充足率閲覧（世帯ごとの公開範囲設定：在庫詳細まで／充足率のみ／非公開）。
- 【贈る】＝ Amazonリンクを開くだけ。受取側に「届いた分を在庫へワンタップ反映」ボタン。
- 防災新聞・イラストテーマ・ジングル・達成バッジは**作らない**（確定）。

### F11 電源・設備モジュール（フェーズ2）
- 機器登録：種別（燃料電池／太陽光／車両給電／ポータブル電源）、出力・容量、メモ。
- 確認タスク管理：停電時機能の有無、専用コンセント位置、周波数設定、自立運転手順など（要確認バッジと連動）。
- 点検リマインド：ポータブル電源の定期充電、車両の**燃料半分ルール**、年1回の給電練習。
- 「電源の使い分け」カード：台風停電（ガス健在）＝太陽光＋燃料電池／ガス停止級の地震＝車両給電＋カセットボンベ、を拠点カードに表示。

## 4. 非機能要件

- コア機能（連絡カード・集合場所・在庫閲覧・安否ボタン）はオフラインで動く。
- 非エンジニアの家族が説明なしで使える：1画面1目的、大きなタップ領域、日本語。
- 無料〜低コスト枠で運用できるサービス選定。家族専用・招待制（一般公開しない）。

## 5. フェーズ計画と受け入れ基準

- **フェーズ1**：F1基本＋F2＋F3＋F4＋F5＋F9（期限・不足通知）を、参考実装をベースに本実装。PWA化・家族アカウント・世帯間同期。
  - 受け入れ基準例：家族のスマホにインストールでき、関西の親が関東の娘世帯の水の充足率を自分の端末で閲覧でき、期限30日前に通知が届く。
- **フェーズ2**：F6＋F7＋F11＋F1の建物属性・バッジ拡張。
- **フェーズ3**：F8＋F10。
- 各フェーズ末に実機確認とふりかえりを行い、次フェーズの要件を見直してよい。

## 6. スコープ外（作らないもの）

自動注文／自動配送手配／公式緊急地震速報の直接受信／通知音カスタム／防災新聞・装飾要素／位置情報の収集／バーコード必須化。
